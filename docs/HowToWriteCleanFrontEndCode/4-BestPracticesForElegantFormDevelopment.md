# Best Practices for Elegant Form development

## 什么是表单组件？

表单组件是有固定的开发范式

- 受控组件
- 非受控组件

受控组件就是组件的内部状态可以通过修改属性值的方式进行控制，受控组件必须满足

- 存在一个 value 的属性
  - 组件的初始值由 value 属性值决定
  - value 属性值变化后组件的内部状态也必须跟着变化
- 抛出一个 onChange 事件
  - 组件内部状态变化后，以抛出 onChange 事件的方式将表单项的最新值转递给父组件

非受控组件就是组件的内部状态由组件自身维护，而不是受到外部传递过来的 props 控制

- 对外提供一个名为 defaultValue 的属性
  - 组件的初始状态由 defaultValue 控制
  - 后续组件状态随着用户交互而进行变，但不通知父组件
- 对外提供一个可以获取组件内部状态的方法
  - 一般通过 ref 方式访问组件内部状态的 value
  - 提供一个 method 方法给外部调用

常见问题(Q & A)

问题 1：充满细节
将表单项提取成组件
问题 2：频繁的 DOM
通过配置驱动渲染
问题 3：表单组件封装不规范
受控组件和非受控组件
问题 4：表单校验不统一
提取常用校验规则到常量中

[config-form](https://github.com/501351981/config-form)

- 高内聚
- 不能为了复用而增加耦合性
  针对业务开发中的复杂表单问题，可以利用分治思想，将复杂表单拆分成多个简单小表单，每个小表单遵循受控组件和非受控组件规范，并额外提供一个 validate 方法。
